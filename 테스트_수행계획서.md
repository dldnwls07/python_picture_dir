## 프로젝트 개요

* **프로젝트명:** 감정 일기장 (Emotion Diary)
* **시스템명:** PyQt5 / Tkinter 두 가지 GUI 백엔드를 지원하는 로컬 데스크톱 일기 애플리케이션 (`main.py --gui qt|tk`)
* **계획된 핵심 구성:**
  - **도메인 계층** (`domain/`): `Diary` 엔티티, `EmotionScore`/`Weather`/`DiaryFilter` 값 객체, 날씨·감정·학점·위치·키워드 다중 조건 필터링 규칙(`Diary.matches_filter`)
  - **애플리케이션 계층** (`application/service/diary_service.py`): 일기 저장/조회/삭제, AI 요약·공감 호출, 비밀번호·필터링·위치 프리셋 오케스트레이션, 비밀 일기장 전용 조회(`get_hidden_diaries`), 날짜별 평균 감정 점수 조회(`get_emotion_scores_by_date`, 기간 옵션 지원) 및 감정 그래프 생성(`generate_trend_chart`)
  - **인프라 계층** (`infrastructure/`): `CSVDiaryRepository`(CSV 원자적 저장), `AIService`(Gemini API 연동), `SecretPasswordStore`(전역 비밀번호 파일 저장/검증), `LocationPresetStore`(위치 프리셋 파일 저장/추가)
  - **엔진 계층** (`engine/`): `AIHelper`(Gemini 요약/공감 호출, 모델 폴백 순서 관리), `WeatherEngine`(Open-Meteo/기상청 연동), `TextProcessor`/`KeywordAnalyzer`(형태소 전처리·워드클라우드), `TrendChartRenderer`(기간별 감정 점수 추이 꺾은선 그래프 PNG 렌더링)
  - **프레젠테이션 계층**: Qt(`app_gui.py`) — `EmotionCalendarWidget`(히트맵 캘린더 + 주간 미니 선그래프 오버레이), `QStackedWidget` 기반 캘린더(MAIN)/편집 페이지 전환. Tk(`app_tkinter.py`) — `EmotionCalendarFrame`(단일 `tk.Canvas` 기반 캘린더 + 미니 선그래프), `pack`/`pack_forget` 기반 페이지 전환
* **작성자:** dldnwls07(기능 구현 예정), konkon3660(저장소 정리·리뷰 예정)
* **테스트 배경:** 본 문서는 아직 개발을 시작하지 않은 감정 일기장 애플리케이션을 대상으로, 각 기능이 요구사항대로 구현되었는지를 검증하기 위한 테스트 범위와 기준을 개발 착수 전에 미리 수립하는 것을 목적으로 합니다. 아래 항목들은 요구사항 논의를 거쳐 확정된 기능 명세이며, 실제 구현 이후 이 계획서의 테스트 항목을 기준으로 검증을 수행합니다.
  1. AI 공감 기능을 저장 버튼에 통합하고 1줄 요약/공감/그림분석 호출을 분리한다
  2. 비밀 일기 비밀번호를 일기별 저장이 아닌 전역 단일 비밀번호(`data/password.txt`)로 관리한다
  3. **비밀 일기장 모드 신설** — 비밀번호 확인 후 숨겨진 일기만 읽기 전용으로 조회 가능한 별도 화면 상태(목록 필터 + 배경색 연출)로 전환한다
  4. **찢기 연출 도입** — 비밀 일기로 저장하면 AI 요약/공감 처리가 시작되기 **전에** 먼저 찢기 연출이 재생되고, 연출이 끝난 뒤 저장·AI 처리가 진행되도록 순서를 정의한다
  5. Gemini 모델 호출 순서를 `gemini-3.5-flash → 3.1-flash-lite → 2.5-flash` 순으로 정의한다
  6. **위치 프리셋 도입** — 위치 입력을 편집 가능한 콤보박스로 제공하고, 저장 시 새 위치를 `LocationPresetStore`에 자동 편입한다
  7. **다중 조건 필터링(`DiaryFilter`) 도입 및 학점(티어) 필터 추가** — 카테고리(날씨/감정)·학점·위치·제목/본문/요약 키워드를 AND 조합으로 검사하고, 일기 목록과 키워드 분석 다이얼로그가 공용 필터 위젯 생성 헬퍼를 공유하도록 설계한다
  8. **UI 대폭 개편** — 좌측 일기 목록·필터 패널을 항상 고정 표시되는 내비게이션으로, 우측 패널을 감정 점수 히트맵 캘린더(MAIN 화면)와 일기 편집 화면 두 페이지짜리 스택으로 구성한다. 캘린더에서 빈 날짜 클릭 시 새 일기 작성, 기존 일기가 있는 날짜 클릭 시 목록 선택과 동일하게 편집 페이지로 전환(비밀 일기는 선택 차단)되도록 한다
  9. **감정 그래프 심화** — 캘린더 히트맵에 다크/라이트 테마별 정밀 색상 팔레트를 적용하고, 캘린더 위에 한 주 단위로 감정 점수를 잇는 미니 선그래프 오버레이를 추가(축 고정 -5~5, 데이터 공백은 끊어서 표현)하며, 기간(시작~종료일) 선택 매크로 뷰 그래프 팝업을 제공한다
  10. 날씨 콤보박스를 하루 1개만 선택하도록 단일화하고, 모든 드롭다운·날짜 선택 팝업이 배경과 명확히 구분되도록 스타일을 정의한다
  11. **일기 목록 스크롤바 안정성** — 마우스오버 등 상호작용으로 항목 텍스트가 늘어나 스크롤바가 나타났다 사라졌다 하는 현상이 없도록, 전문은 툴팁으로만 노출하고 항목 크기는 고정되도록 설계한다

  아울러 앞으로 논의될 추가 기능(해시태그, 일기장 안에서의 이점 제공 등)을 안전하게 얹을 수 있는 회귀 테스트 기반을 마련하는 것도 목적에 포함합니다.

## 비고 (예상 리스크 및 유의사항)

개발 착수 전 시점에서, 선택한 기술 스택과 기능 요구사항의 특성상 미리 유의해야 할 사항을 정리합니다.
아래 항목은 아직 발생이 확인된 결함이 아니라, 구현 단계에서 발생 가능성이 있어 설계·구현 시 주의가
필요한 위험 요인입니다.

- 콘솔 로그 문자열에 이모지를 포함시킬 경우, Windows 콘솔의 기본 코드페이지(cp949)에서
  `UnicodeEncodeError`가 발생할 수 있으므로 로그 출력 인코딩을 명시적으로 처리하거나 이모지 사용을
  최소화하는 방향을 검토해야 한다.
- Tkinter는 같은 부모 위젯 안에서 `grid`와 `pack` 지오메트리 매니저를 함께 사용하면 `TclError`가
  발생하므로, 화면 구성 시 위젯마다 지오메트리 매니저를 통일해서 사용해야 한다.
- Tk에서 비밀 일기장 모드의 배경색 연출을 공용 ttk 스타일(`"TFrame"`)을 직접 변경하는 방식으로
  구현할 경우, 모드 진입 중 다른 다이얼로그(키워드 분석 등)를 동시에 띄우면 그 배경색도 함께
  바뀔 수 있어 설계 시 스타일 적용 범위를 분리하는 방안을 검토해야 한다.
- PyQt5의 `QDialog`는 부모 위젯의 스타일시트를 자동으로 상속하지 않으므로(별도의 최상위 창으로
  취급됨), 다이얼로그마다 개별적으로 스타일을 지정하면 하나라도 누락된 드롭다운/날짜 선택 팝업이
  OS 기본 색상으로 표시되어 다크 테마와 대비가 낮아질 수 있다. `QApplication` 전역 스타일시트로
  공통 스타일을 적용하는 방식을 우선 검토해야 한다.
- Qt 일기 목록에서 마우스오버 시 항목 텍스트를 전문으로 확장하는 방식으로 구현하면, 항목 높이
  변화로 인해 목록 스크롤바가 나타났다 사라졌다 하는 현상이 발생할 수 있다. 전문은 툴팁으로
  노출하고 항목 자체의 크기는 고정하는 방식을 우선 설계안으로 검토해야 한다.
- 캘린더 위 주간 미니 선그래프(Qt)를 `QCalendarWidget` 내부의 비공식 구조(날짜 그리드 뷰)에 의존해
  구현할 경우, PyQt5 마이너 버전에 따라 내부 위젯 트리가 달라져 오버레이가 표시되지 않을 위험이
  있다. 해당 구조를 찾지 못하는 경우에도 앱이 죽지 않고 히트맵 색상만은 정상 표시되도록 방어적으로
  구현해야 한다.
