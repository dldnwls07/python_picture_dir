## 1. 감정 쓰레기통이라는 컨셉에 맞게 "숨기기"라는 기능의 이름을 일기 찢기로 바꾸는거야. — ✅ 구현 완료
(원래 `PLAN.md` 4번)

**구현 내용**

- **비밀번호 검증**: `SecretPasswordStore.verify_password()` 추가(평문 비교). `DiaryService.verify_secret_password()`/`get_hidden_diaries()` 추가.
- **비밀 일기장 모드 (양쪽 GUI 공통)**:
  - "🔒 비밀일기 찾기" 버튼 → 비밀번호가 아직 없으면 안내, 있으면 매번 비밀번호 입력을 요구(4-2-4) → 통과 시 모드 진입.
  - 진입하면 목록이 숨겨진 일기만 보여주도록 전환(`get_hidden_diaries`), 필터 콤보박스는 그대로 유지(4-2-2).
  - "새 일기"/"저장" 버튼 비활성화(4-2-3, 4-2-2 저장 금지), 편집 폼 전체를 읽기 전용으로 잠금. **삭제 버튼은 그대로 허용**(4-2-2).
  - "🚪 나가기" 버튼(4-2-1, 위치는 임시 배치 — `PLAN.md` 7번에서 정식 배치 예정) 클릭 시 일반 모드로 복귀.
  - 모드 진입 중에는 좌우 패널 배경색이 두 색 사이를 천천히 왕복하는 연출(4-2-5)이 재생됨.
- **찢기 연출(4-1)**: 비밀 일기를 저장하면 AI 다이얼로그가 닫힌 직후 재생됨.
  - **PyQt**: `rightPanel`을 스냅샷 → 지그재그 선으로 위/아래 두 조각으로 나눠 반대 방향으로 날아가며 페이드아웃(`QPropertyAnimation`/`QParallelAnimationGroup`).
  - **Tkinter**: 결정한 대로 단순 대체 연출 — 📄💨 이모지를 화면 중앙에 잠깐 띄웠다 사라지게 함.
- **색상 연출 구현 차이**: Qt는 `QVariantAnimation` + `finished` 시그널에서 방향을 뒤집는 핑퐁 방식(부드러운 사인 이징, 6초 단위). Tk는 `after()` 루프로 같은 핑퐁을 흉내내되(ttk `"TFrame"` 스타일의 배경색을 매 틱마다 갱신), 애니메이션 프레임워크가 없어 더 단순하게 구현함.

**알려진 사소한 제약**
- Tk의 색상 연출은 공용 ttk 스타일(`"TFrame"`)을 직접 바꾸는 방식이라, 비밀 일기장 모드 중에 다른 다이얼로그(예: 키워드 분석 창)를 띄우면 그 다이얼로그의 배경색도 같이 바뀜 — 동시에 열어보는 경우가 드물 것이라 판단해 우선순위를 낮게 두고 그대로 둠.
- "나가기"/"비밀일기 찾기" 버튼 위치는 버튼 줄 오른쪽에 임시로 배치함 — `PLAN.md` 7번(UI 개편) 때 정식 위치를 정하기로 함.

**스모크 테스트 결과**: 정상 진입/읽기전용 잠금/목록 전환/나가기 복귀/색상 애니메이션 시작·정지/찢기 연출 재생-콜백 호출까지 Qt·Tk 양쪽에서 직접 호출로 확인함. 유닛 테스트 20개 중 18개 통과(나머지 2개는 이번 작업과 무관한 기존 cp949 콘솔 인코딩 이슈).
