### 4-1. 찢기 연출 — ✅ D번(실제 종이가 찢어지는 지그재그 절단 효과)로 결정

**구현 방향**
- 편집 폼(또는 저장 완료 시점의 스냅샷)을 이미지로 캡처 → 지그재그 선을 기준으로 위/아래(또는 좌/우) 두
  조각으로 나눔 → 각 조각을 반대 방향으로 밀어내며 사라지게 함.
- **PyQt**: `QWidget.grab()`으로 스냅샷 → 지그재그 마스크(`QPainterPath` + `QPixmap` 알파 마스킹)로 두 장의
  `QPixmap` 생성 → 각각 `QLabel`에 얹고 `QPropertyAnimation`으로 반대 방향 이동 + opacity 감소.
- **Tkinter**: `Canvas` 기반으로 비슷하게 흉내낼 수 있으나, 위젯 스냅샷 캡처(Pillow의 `ImageGrab` 등)와
  지그재그 마스킹을 직접 구현해야 해서 Qt보다 작업량이 훨씬 큼.

**남은 확인 필요 사항**
- Tk 쪽도 Qt와 동일한 수준(진짜 지그재그 찢기)으로 구현할지, 아니면 Tk는 더 단순한 대체 연출(예: 이전에
  제안했던 흔들림+이모지)로 갈지 — 작업량 차이가 크다는 점을 감안해서 결정 필요.

### 4-2. 비밀 일기장 UI — 세부 사항 확정

| 항목 | 확정된 사항 |
| --- | --- |
| **4-2-1 나가는 방법** | "나가기" 버튼을 새로 만들어서, 클릭하면 원래 화면(일반 목록 + 원래 테마)으로 복귀. |
| **4-2-2 필터/선택 동작** | 필터 콤보박스(`filterComboBox`)는 그대로 사용 — 비밀 일기 목록에도 동일한 날씨/감정 필터 적용. 다만 목록에서 항목을 선택해도 **읽기 전용**으로만 보여줌(수정 불가). |
| **4-2-3 새 일기 작성** | 비밀 일기장 모드에서는 "새 일기" 작성 자체를 제한함(버튼 비활성화 등). |
| **4-2-4 비밀번호 재확인** | 모드에 들어갈 때마다("왔다갔다 할 때마다") 매번 비밀번호를 다시 요구함 — 한 번 인증했다고 계속 유지되지 않음. |
| **4-2-5 색상 연출** | 고정된 배색으로 한 번만 바뀌는 게 아니라, **불안한 느낌을 주기 위해 색깔이 계속(지속적으로) 바뀌는** 연출로 구성. |

**추가로 정리가 필요해 보이는 사항 (남은 질문)**
1. **읽기 전용의 범위**: "새 일기 작성 제한"은 명시하셨는데, 이미 있는 숨겨진 일기에 대한 **저장(수정)/삭제
   버튼도 비활성화**해야 하는지 확인 필요 — "읽기 전용으로 읽을 수만 있게"라는 문구로 보아 저장/삭제도 막는
   쪽이 맞아 보이지만, 명시적으로 확인하고 싶음.
2. **"나가기" 버튼 위치**: 메인 화면 어디에 둘지 — `PLAN.md` 7번(UI 개편)에서 레이아웃을 다시 짤 때 같이
   자리를 잡는 게 좋아 보임.
3. **색깔이 "계속 바뀌는" 방식의 구체적인 구현**:
   - Qt는 `QTimer`로 일정 주기마다 스타일시트의 색상 값을 갱신하거나, `QPropertyAnimation`을 색상값에
     걸어 무한 반복(loop)시키는 방식이 자연스러움. Tk는 `after()`로 동일한 루프를 직접 구현해야 함.
   - **참고**: 색이 너무 빠르고 강하게 깜빡이면 눈이 피로하거나 일부 사용자에게 불편할 수 있어서,
     "계속 바뀌지만 부드럽게(색상환을 천천히 도는 정도)" 정도로 완급 조절하는 걸 권장 — 원하시는 변화
     속도/강도가 있다면 알려주시면 그대로 반영하겠습니다.
