## 1. 일기 목록에서 한 줄 요약이 화면 밖으로 넘어가 잘리는 문제 — ✅ 구현 완료 (5번 방안, 툴팁 → 호버 확장으로 변경)

**적용한 방안**: 5번(말줄임 미리보기 + 마우스 오버 시 전문 확장)

**변경 내용**
- `diary_categories.py`에 `truncate_summary(text, limit=18)` 추가 — 18자 넘으면 잘라서 `...` 붙임.
- **`app_gui.py`(PyQt)**: 처음엔 툴팁으로 구현했다가, "툴팁이 아니라 항목 자체가 늘어나며 전문이
  보이게" 요청에 맞춰 변경함.
  - `diaryListWidget`에 `setWordWrap(True)` + `setMouseTracking(True)` 적용.
  - `itemEntered` 시그널로 마우스가 올라간 항목을 감지해 `item.setText(전체_텍스트)`로 바꿔치기 →
    word-wrap 덕분에 항목 높이가 늘어나면서 전문이 그대로 보임.
  - `viewport()`에 이벤트 필터를 달아 `QEvent.Leave`(목록 밖으로 마우스가 나감)를 감지해서 원래
    미리보기 텍스트로 되돌림. 다른 항목으로 마우스가 옮겨가면 그 시점에 이전 항목도 자동으로 복구됨.
  - 항목별로 `Qt.UserRole+1`(전문), `Qt.UserRole+2`(미리보기)에 두 텍스트를 저장해두고 필요할 때
    꺼내 씀 — 원본 `Diary` 재조회 없이 즉시 전환 가능.
- **`app_tkinter.py`(Tk)**: 목록 항목은 여전히 잘린 미리보기만 표시. **`tk.Listbox`는 항목별로
  다른 위젯/가변 높이/word-wrap을 지원하지 않는 구조**라, PyQt와 동일한 "호버 시 항목이 늘어나는"
  효과를 지금 구조 그대로는 구현할 수 없음. 대신 기존처럼 항목을 선택하면 상태바에 전체 요약이
  이어붙어(`· 📝 {summary}`) 표시되는 방식을 유지함 (선택 기반, 호버 기반은 아님).

**남겨둔 사항 (당장 처리 안 함)**
- Tk에서도 진짜 "호버 시 확장"을 구현하려면 `Listbox`를 포기하고 스크롤 가능한 `Frame` + 개별
  `Label`(wraplength 지정) 조합이나 `Treeview`로 갈아엎어야 함 — `PLAN.md` 7번(UI 개편) 때 같이
  검토하기로 함.
- 말줄임 기준 글자 수(18자)는 실측 없이 임의로 정한 값 — 실제 폰트/패널 폭 기준으로 나중에 조정 가능.
