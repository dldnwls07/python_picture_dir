## 1. 프로젝트 개요

* **프로젝트명:** 감정 일기장 (Emotion Diary)
* **목적:** 사용자가 매일의 감정·날씨·그림을 기록하고, AI(Gemini API)의 자동 요약·공감·그림 분석을 받아볼 수 있는 로컬 데스크톱 일기 애플리케이션을 구현한다. PyQt5/Tkinter 두 가지 GUI 백엔드를 모두 지원한다(`main.py --gui qt|tk`).
* **배경:** 감정을 있는 그대로 기록하고 필요하면 완전히 숨길 수 있는 "감정 쓰레기통" 컨셉에서 출발했으며, 초기 프로토타입 이후 AI 연동·비밀 일기장·필터링 등 기능을 단계적으로 추가하는 방식으로 진행 중이다.
* **핵심 구성:**
  - **도메인 계층** (`domain/`): `Diary` 엔티티, `EmotionScore`/`Weather`/`DiaryFilter` 값 객체
  - **애플리케이션 계층** (`application/service/diary_service.py`): 저장/조회/삭제, AI 요약·공감 호출, 비밀번호·위치 프리셋·필터링 오케스트레이션
  - **인프라 계층** (`infrastructure/`): `CSVDiaryRepository`(CSV 원자적 저장), `AIService`(Gemini API 연동), `SecretPasswordStore`, `LocationPresetStore`
  - **엔진 계층** (`engine/`): `AIHelper`, `WeatherEngine`(Open-Meteo/기상청 연동), `TextProcessor`/`KeywordAnalyzer`
* **참여 인원 및 역할:**

## 3. 프로젝트 범위

### 범위 내 (In Scope)
- PyQt5 / Tkinter 양쪽 GUI에서 동일하게 동작하는 일기 작성·조회·삭제·필터링 기능
- Gemini API 기반 AI 1줄 요약 / 공감 멘트 / 그림 분석
- 비밀 일기장(찢기 연출 + 읽기 전용 열람) 기능
- 위치/날씨/감정 프리셋 및 다중 조건 필터링
- `PLAN.md` 7~10번(UI 개편, 감정 그래프, 해시태그, 이점 제공 기능)

### 범위 외 (Out of Scope, 현재 기준)
- 클라우드 동기화/멀티 디바이스 지원 (로컬 CSV 기반 단일 사용자 앱)
- 모바일 앱 버전
- 비밀번호 해시화 등 보안 강화 작업(현재 `data/password.txt` 평문 저장 — 별도 결정 전까지 범위 외로 둠)
