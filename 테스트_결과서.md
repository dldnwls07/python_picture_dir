## 테스트 개요

* **목적:** `테스트_수행계획서.md`에서 정의한 테스트 항목을 기준으로, 구현이 완료된 감정 일기장 애플리케이션이 요구사항대로 동작하는지 검증하고 결과를 기록한다.
* **테스트 대상:** `PLAN.md`에 정의된 1~10번 기능 전체 (AI 공감 자동화, 비밀번호 전역화, 비밀 일기장 모드, 찢기 연출, AI 모델 호출 순서, 위치 프리셋, 다중 조건 필터링, UI 대폭 개편, 감정 그래프, 해시태그, 일기장 안에서의 이점 제공)
* **테스트 방법:**
  - 자동화된 단위/통합 테스트: `python -m unittest discover -s tests` (`tests/` 디렉터리, 총 9개 테스트 모듈)
  - 시스템/인터페이스 테스트: 체크리스트 기반 수동 확인(Qt는 `QT_QPA_PLATFORM=offscreen` 헤드리스 실행 병행, Tk는 로컬 디스플레이로 확인)

## 발견된 결함 및 조치 내역

테스트 수행 과정에서 실제로 발견되어 조치된 결함과, 조치되지 않고 남아 있는 이슈를 함께 기록합니다.

| 번호 | 결함 내용 | 발견 시점 | 심각도 | 조치 내용 | 상태 |
| :---: | :--- | :--- | :---: | :--- | :---: |
| D-1 | Qt `QDialog`(키워드 분석·감정 그래프 등)가 메인 창의 스타일시트를 상속하지 않아, 일부 드롭다운/날짜 선택 팝업이 OS 기본 색상으로 표시되어 다크 테마와 대비가 낮음 | 인터페이스 테스트 중 | 중 | `QApplication` 전역 스타일시트를 추가해 모든 다이얼로그에 공통 스타일 적용 | 조치완료 |
| D-2 | Qt 일기 목록에서 마우스오버 시 항목 텍스트가 전문으로 확장되며 항목 높이가 변해 목록 스크롤바가 나타났다 사라졌다 함 | 인터페이스 테스트 중 | 하 | 전문은 툴팁으로만 노출하고 항목 텍스트/높이는 고정되도록 변경 | 조치완료 |
| D-3 | Qt 앱 최초 실행 시 `QListWidget`이 0번 항목을 내부적으로 자동 선택해 편집 페이지로 잘못 전환됨 | 통합 테스트 중 | 중 | 최초 1회의 자동 선택 이벤트를 무시하는 플래그를 추가하고, 목록 선택도 함께 초기화 | 조치완료 |
| D-4 | `engine/ai_helper.py`의 예외 로그에 포함된 이모지(⚠️)가 Windows 콘솔 기본 코드페이지(cp949)에서 `UnicodeEncodeError`를 발생시켜 관련 단위 테스트 2건이 이 환경에서 실패로 표시됨 | 단위 테스트 중 | 하 | 기능 자체의 결함이 아니며, 로그 인코딩 처리 방식 개선이 필요 | 미해결 |
| D-5 | Tk 비밀 일기장 모드의 배경색 연출이 공용 ttk 스타일(`"TFrame"`)을 직접 변경하는 방식이라, 모드 진입 중 다른 다이얼로그를 동시에 띄우면 배경색이 함께 바뀜 | 인터페이스 테스트 중 | 하 | 낮은 우선순위로 별도 결함으로만 기록, 조치 보류 | 미해결(낮은 우선순위) |

  낮거나 환경 의존적인 사항으로 남겨두었다.
* 위 결과를 근거로, 본 프로젝트는 계획된 기능 범위 내에서 배포 가능한 상태로 판단한다.
