## 프로젝트 개요

* **프로젝트명:** 감정 일기장 (Emotion Diary)
* **시스템명:** PyQt5 / Tkinter 두 가지 GUI 백엔드를 지원하는 로컬 데스크톱 일기 애플리케이션 (`main.py --gui qt|tk`)
* **핵심 구성:**
  - **도메인 계층** (`domain/`): `Diary` 엔티티, `EmotionScore`/`Weather` 값 객체, 날씨·감정 필터링 규칙
  - **애플리케이션 계층** (`application/service/diary_service.py`): 일기 저장/조회/삭제, AI 요약·공감 호출, 비밀번호·필터링 오케스트레이션
  - **인프라 계층** (`infrastructure/`): `CSVDiaryRepository`(CSV 원자적 저장), `AIService`(Gemini API 연동), `SecretPasswordStore`(전역 비밀번호 파일)
  - **엔진 계층** (`engine/`): `AIHelper`(Gemini 요약/공감 호출), `WeatherEngine`(Open-Meteo/기상청 연동), `TextProcessor`/`KeywordAnalyzer`(형태소 전처리·워드클라우드)
* **작성자:** dldnwls07(기능 구현), konkon3660(저장소 정리·리뷰)
* **테스트 배경:** 본 문서는 이미 구현된 감정 일기장 애플리케이션을 대상으로, 최근 반영된 주요 변경사항(① AI 공감 기능을 저장 버튼에 통합하고 1줄 요약/공감/그림분석 호출을 분리한 것, ② 비밀 일기 비밀번호를 일기별 저장에서 전역 단일 비밀번호(`data/password.txt`)로 전환한 것)이 기존 기능을 손상시키지 않았는지 검증하고, 앞으로 남은 작업(`PLAN.md` 3~7번: 위치 프리셋, 통합 필터 함수, UI 개편, 찢기 연출·비밀 일기장 UI)을 안전하게 얹을 수 있는 회귀 테스트 기반을 마련하는 것을 목적으로 합니다.

## 비고 (현재 알려진 이슈)

- `engine/ai_helper.py`의 `print()` 로그 문에 포함된 이모지(⚠️)가 Windows 콘솔 기본 코드페이지(cp949)에서 `UnicodeEncodeError`를 발생시켜, `test_network_failure_raises_runtime_error` / `test_malformed_json_response_raises_runtime_error` 두 테스트가 이 환경에서는 실패로 표시됨 (기능 자체의 결함은 아님).
- `app_tkinter.py`의 `_init_ui()`에서 `grid`/`pack` 지오메트리 매니저를 같은 부모 위젯에 함께 사용하고 있어, 이 Tcl/Tk 버전에서 `TclError: cannot use geometry manager pack inside . which already has slaves managed by grid`가 발생함 — 리팩터링 이전부터 존재하던 결함으로 확인됨.
